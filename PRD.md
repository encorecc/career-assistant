# 基于豆包大模型的职场对话画像与交涉建议 H5（PRD）

## 1. 概述
- 名称：职场对话画像与交涉建议 H5
- 目标：用户上传 5 张左右的职场对话截图，调用豆包大模型（doubao-seed-1.6-thinking），生成对交流对象的画像与交涉建议，帮助用户在特定目标下达成专业的同事关系，不被低估、不被忽悠（不保证结果、仅提供建议）。
- 平台：移动端 H5
- 模型与文档：Volcengine Ark Responses API；模型 doubao-seed-1.6-thinking；文档 https://www.volcengine.com/docs/82379/1399008

## 2. 目标与范围
- 主要目标：
  - 从职场对话截图中提取对方的角色定位、诉求与优先级、沟通风格与影响力。
  - 产出合宜的交涉策略与关键措辞，帮助用户在职场场景中高效推进目标。
- 非目标：
  - 不进行身份识别、人身攻击、跟踪或任何侵犯隐私的行为。
  - 不承诺撮合成功；输出为参考建议。

## 3. 用户与场景
- 角色：发起分析的用户（上传者）
- 前置：已获取交流对象的职场对话截图（不少于 5 张）
- 场景：用户希望通过历史对话内容了解对方诉求与风格，并获得专业的交涉策略与关键措辞

## 4. 关键功能需求
1) 上传与校验
- 支持上传 5–10 张图片（PNG/JPG），单张 ≤10MB，合计 ≤80MB
- 批量选择、上传进度与失败重试
- 本地预览、可删除和重新选择；去重提示
- 选填补充信息：目标对象大致年龄范围、性别倾向、用户与对方的关系阶段（陌生/点过赞/私聊过等）

2) 分析与生成
- 将图片及可选补充文本传给豆包模型进行多模态分析
- 系统 Prompt：见第 14 节，随请求注入
- 分析维度（示例）：
  - 角色与倾向：同事/领导/跨部门、强势/务实/风险偏好
  - 诉求与优先级：指标、资源、风控、时间表、政治考量
  - 沟通风格与影响力：直接/暗示、共识驱动/权威驱动、话语权
  - 场景目标与阻力：关键阻点、可换位思考点、潜在误解
- 输出建议：
  - 开场与关键措辞模板（可复制）
  - 交涉策略与议题清单（分主次与时序）
  - Do / Don’t 清单（应做/不宜做）
  - 风险提示与备选路径
  - 置信度与免责声明

3) 同页报告呈现
- 在上传页内联展示：画像摘要 + 交涉建议 + 风险提示
- 支持复制消息、保存为图片/分享（可选）
- 二次优化入口：用户反馈标签（有用/一般/不适合），用于再优化

4) 留存与反馈（MVP 可选）
- 是否允许临时缓存分析结果（默认不持久存储）
- 反馈收集：满意度评分与是否采纳建议（例如点击复制）

## 5. 非功能与合规
- 隐私与伦理：
  - 不收集敏感身份信息；不进行人脸识别或具体身份推断
  - 不鼓励跟踪、骚扰或操控式建议；输出保持尊重与合宜
  - 图片默认仅用于即时分析，不持久存储；用户可选择授权保存
- 安全：
  - 全站 HTTPS；后端鉴权与速率限制；内容安全与合规审查
  - API Key 使用环境变量 ARK_API_KEY，不写入客户端
- 性能：
  - 首次分析目标 ≤20 秒（视模型与网络而定）
  - 上传断点续传与失败重试

## 6. 交互流程（MVP）
- 上传页：选择图片（≥5）→ 选填补充信息 → 提交
- 同页分析与结果：展示进度与提示（可取消）；画像摘要、建议列表、风险提示内联展示；支持复制开场消息
- 反馈：满意度与采纳（点击复制视为采纳）

## 7. 输出结构（建议）
```json
{
  "summary": {
    "traits": ["角色与倾向示例：领导、共识驱动"],
    "interests": ["关注点示例：资源分配、时间表稳定性"],
    "communication_style": "沟通风格示例：直接且注重数据",
    "values": ["职场价值示例：结果导向、风险控制"]
  },
  "recommendations": {
    "openers": ["开场示例：先对共同目标对齐，再给出两种可选方案"],
    "topics": ["议题示例：资源与时间优先级、风险缓解措施"],
    "activities": ["行动建议示例：先小范围试点，再滚动扩张"],
    "dos": ["数据支撑观点", "给出备选路径与妥协空间"],
    "donts": ["情绪化表述", "在公众场合强硬争执"]
  },
  "risks": ["阻力示例：跨部门利益冲突、对风险敞口的担忧"],
  "confidence": 0.72,
  "disclaimer": "建议仅供参考，请遵循职场伦理与合规规范"
}
```

## 8. 文案要点
- 上传页提示：至少 5 张截图；不含敏感隐私；仅用于即时分析
- 同页提示：生成约需 10–20 秒；网络不佳时可能更久
- 免责声明：不保证结果、请尊重对方边界与隐私
- 行动建议文案：专业、数据化、以共同目标为中心，避免情绪化与强硬逼迫

## 9. 技术方案（概述）
- 架构：单页 H5（如 Vue/React/原生）+ 后端中转服务（Node/Python/Go 皆可）
- 模型调用：Volcengine Ark Responses API
  - Base URL: https://ark.cn-beijing.volces.com/api/v3
- 模型：doubao-seed-1.6-thinking（默认启用深度思考）
  - 鉴权：Authorization: Bearer $ARK_API_KEY（服务端）
  - 输入：图片（URL 或 Base64）+ 用户补充文本（可选）
  - 输出：结构化文本（按第 7 节的 JSON 结构解析并呈现）
- 上传处理：
  - 前端压缩与去重；支持断点续传
  - 服务端做病毒与内容安全检查（可接入云服务）
- 存储策略：
  - 默认仅临时缓存分析过程数据；结果不持久化
  - 用户授权后可保存结果以便复用（合规前提）

## 10. API 设计（示例）
- POST /api/analyze
  - 请求体：
    - images: [ { type: "base64" | "url", value: "..." } ]（≥5）
    - meta: { goal?: "推进资源|明确职责|争取支持", context?: "会议纪要/群聊截屏线索" }
  - 响应体：第 7 节所示的 JSON
- 失败码：
  - 400 参数不足或格式错误
  - 413 文件过大
  - 429 频率受限
  - 500 服务异常

## 11. 成功指标（MVP）
- 报告生成成功率 ≥95%
- 平均分析时长 ≤20 秒
- 复制开场措辞的点击率 ≥40%
- 用户满意度（≥4/5）占比 ≥60%

## 12. 里程碑
- M1（设计）：完成交互稿与提示词初稿
- M2（开发）：完成单页上传、后端中转、模型调用与同页内联结果
- M3（内测）：小流量验证与合规审查
- M4（上线）：稳定发布与持续优化

## 13. 风险与对策
- 模型误判或偏见：加入风险与免责声明；允许用户反馈修正
- 侵入式建议风险：严格提示词规范与内容审查，避免冒犯与操控
- 合规与版权：提示用户确保上传素材的合法性与授权
- 性能与费用：图片压缩与并发控制；对调用次数进行限流与监控

## 14. 提示词（系统 Prompt）
- 系统 Prompt（上传图片时注入）：你是一个职场交涉的高手，对于职场同事或领导的心理和外在表现，有非常强的洞察，也有一套很厉害的处理复杂politics的技巧！擅长于输出简短但有效的分析和建议。
- 用户输入结构：图片（≥5）+ 可选补充信息（目标与场景线索）
- 输出要求：请按第 7 节的 JSON 结构返回，并在建议中体现与图片线索的映射依据。
